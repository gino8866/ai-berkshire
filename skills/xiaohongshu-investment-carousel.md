# 小红书投研图轮播

把结构化单公司投研底稿转换为普通投资者能在手机端扫读、又可复核数据口径的逐页轮播稿。

**核心原则：固定研究骨架，按数据关系灵活选版；先给判断，再给证据，最后给边界。**

## 输入与边界

开始前读取完整底稿和全部参考图，不要只读摘要或结论。

- 默认受众：**小红书普通投资者**。仅在用户明确指定时切换为专业投研、行业从业者或其他人群。
- 默认页数：正文 9–12 页（不含封面）。封面不计入页码，正文从第 1 页开始计数。按材料复杂度调整，不为凑页数拆散逻辑。
- 先确认当前日期、报告研究日期和数据截止日；把当前日期作为判断市场数据是否过期的基准。
- 本 Skill 产出逐页图轮播脚本，不直接生成 PNG/JPG。
- 缺少研究底稿时，先使用相应研究与财务数据 Skill；不要用常识补造事实。
- 不把行业研究、主题投资或多公司篮子强行套入单公司结构。遇到这类输入时立即停止本流程，只简短说明本 Skill 不适用，并改用相应行业研究或内容设计 Skill。**不得继续输出页面规划、占位稿、通用骨架或“数据待补”的行业轮播。**
- 用户提供参考图时，提取画布、色板、字体气质、信息层级与密度、图表语言、批注方式和页底节奏；学习结构，不复制参考公司的内容。
- 参考图为封面时，拆解为“系列标题 + 一句话结论 + 公司代表性手绘图 + 数据卡”的层级。数据卡只保留印证结论的 2–4 组；主体插画可抽象表达公司业务，不伪造 Logo、产品截图或品牌资产。
- 没有参考图时，先给 2–3 种叙事方向并推荐一种，再展开完整稿。

## 工作流

### 1. 建立证据字段表

先检测报告是否具有“元信息 + 商业模式与竞争壁垒 + 财务质量 + 成长与估值 + 风险 + 综合结论 + 数据来源”的固定结构。结构匹配时按下表做确定性字段映射；字段映射用于防漏，不等于一章一页。

| 来源章节 | 内容职责 | 优先承载页面 |
|---|---|---|
| 元信息块 | 研究日期、市值、定位、初筛评级、数据截止口径 | 封面 |
| 1.1 主营业务 | 收入结构、基本盘、新变量 | 业务结构 |
| 1.2 行业地位 + 1.3 龙头差距 | 国内/全球位置、已做到与仍有差距 | 竞争位置 |
| 1.4 护城河 | 技术、客户、规模、政策及脆弱性 | 护城河 |
| 2.1 营收利润 + 2.3 订单/合同负债/存货 | 经营趋势、可见度、异常信号 | 经营趋势 |
| 2.2 现金流 + 2.4 资产负债表 + 2.5 研发/ROE | 利润质量、杠杆、投入效率 | 财务质量 |
| 3.1 增长驱动 + 3.2 行业空间 | 基本盘、第二曲线、远期想象及不确定性 | 成长性 |
| 3.3 当前估值 + 3.4 合理区间 | 当前倍数、机构预期、情景和隐含预期 | 估值 |
| 4.1–4.5 多维风险 + 4.6 最大下行 | 风险/对冲、量化压力测试 | 风险与跟踪 |
| 5.1–5.4 综合结论、是否跟踪、跟踪指标、估值锚 | 研究倾向、长短期判断、验证仪表、估值依赖 | **正文第 1 页结论** |
| 数据来源与局限性 | 来源分类、A/E/估算口径、免责声明 | 最后一页 |

若报告不使用上述章节编号，回退到语义提取。识别以下字段组：

| 字段组 | 必提取内容 |
|---|---|
| 元信息 | 研究日期、数据截止日、市值、公司定位、初筛结论 |
| 业务 | 收入结构、基本盘、新变量、业务增速与毛利率 |
| 竞争 | 国内/全球位置、龙头差距、差距是否缩小 |
| 护城河 | 技术、客户、规模、政策及每项脆弱性 |
| 经营 | 3–5 年营收利润、最新季度、订单、合同负债、存货和异常信号 |
| 财务 | 现金流/净利润、杠杆、应收、研发投入、ROE 或投入效率 |
| 成长 | 基本盘、第二曲线、远期想象、行业空间和各自不确定性 |
| 估值 | 当前倍数、历史分位、机构预期、同行、三情景和隐含预期 |
| 风险 | 风险等级、触发条件、对冲论据、量化最大下行情景 |
| 结论 | 长期逻辑、短期问题、跟踪指标、估值锚和研究倾向 |
| 来源 | 公司披露、行业数据、市场/预期、A/E/估算/情景口径 |

字段缺失时写明“数据不足”或“来源缺口”。新增事实必须按项目研究规则核验；估算值明确标“估算”。

**数据纪律高于模板完整性。** 不得为凑齐页数、估值锚、跟踪阈值或下行情景而编造数字。缺少可复核输入时，保留对应页面位置，写清缺失字段与补数要求，并把整稿标记为“数据待补，不可发布”；不要用未经验证的假设填空。

### 2. 确定叙事骨架

默认顺序：

`封面冲突 → 统一结论 → 业务 → 竞争/护城河 → 成长 → 经营/财务 → 估值 → 风险与跟踪 → 来源与声明`

强制约束：

1. **正文第 1 页固定为统一结论页**（封面不计入页码）。合并长期判断、短期问题、是否持续跟踪、关键跟踪指标、估值锚和估值依赖关系；后续页面负责证明它。
2. **最后一页固定为来源、口径和免责声明。** 不得用“适合谁”“总结”或互动引导替代。
3. 中间页面允许合并、拆分和重排，以公司的最大投资矛盾为主线。
4. 不得省略最大下行风险、跟踪指标或估值锚的位置。底稿数据充分时量化；数据不足时明确缺口和补数要求，不得强行给数。
5. 每页只回答一个投资问题；**页尾判断只收束当前页的结论，不得提问、预告或引出下一页内容。**

结论可以有研究倾向，例如“提示风险”“持续关注”“具有好股票 + 好价格特征”“叙事确实好，但估值对叙事的依赖程度极高”。

不得输出“买入、卖出、持有、加仓、减仓、建仓、清仓”等明确交易或仓位信号。把底稿中的交易时机改写为估值锚、观察区间、关键变量或叙事兑现条件。

写完整逐页稿前，先给出建议页数、页面顺序和信息密度，并等待用户确认。用户已经明确指定或确认这些项目时，不重复提问。

### 3. 先判断数据关系，再选择布局

#### 直接出图的布局边界

逐页脚本必须让设计师或图像生成器知道“画什么数据、如何比较、突出什么”，但**布局只描述区块关系，不写坐标、像素、百分比、固定宽高或留白数值**。例如写“主图居中、信息带紧贴图表下方、页尾置底”，不要写“主图占 56%”或“卡片距右侧 24 px”。

- 数据图、卡片和批注必须围绕同一视觉中心成组出现；不得把无配对内容的卡片孤立放在右下角或左下角，造成大片空白。
- 主图下方若需要补充经营代价、资产负债或投入数据，优先使用横贯页面的信息带或并列摘要，而不是角落悬浮卡。
- 只有两个图回答互补问题时才并列；同一结论的重复图表应合并或删减。

| 主数据关系 | 首选布局 | 常见用途 | 禁忌 |
|---|---|---|---|
| 部分与整体 | 结构条 + 紧凑表格 | 收入、地区、客户、成本结构 | 切片过多；表格重复图表全部数字 |
| 两个维度互相矛盾 | 左右或上下对照 | 长期优势/短期压力、利润/现金流 | 两栏失衡；重复同一事实 |
| 因果链或生态闭环 | 中轴流程 + 两侧证据 | 商业模式、护城河、产业链、增长传导 | 把产品清单伪装成流程 |
| 时间趋势出现异常 | 主趋势图 + 侧边异常卡 | 增收不增利、季度急刹车、毛利率拐点 | 把季度与全年混成同一口径 |
| 历史变化与横向差异 | 左趋势 + 右比较 | 利润结构、业务毛利率、份额变化 | 维度超过四个；没有业务解释 |
| 当前估值依赖未来 | 当前快照 + 未来要求 | 历史分位、机构预期、隐含增长 | 把预期写成实际；把低分位写成低风险 |
| 阶段或成熟度不同 | 阶梯、漏斗或成熟度轴 | 基本盘、第二曲线、远期想象 | 用产品名替代阶段判断；超过三层 |
| 结论需要后续验证 | 分栏判断 + 跟踪仪表 | 正文第 1 页结论、关键变量和风险信号 | 指标无阈值；超过四个仪表 |
| 来源与责任边界 | 分级来源 + 口径区 + 声明区 | 最后一页 | 只写机构名，不写日期和口径 |

一页同时出现三种以上关系时，选出唯一主关系；其他内容降为辅证、移页或删除。不要并列两个同权重主图。

### 4. 从九类母版中选用

母版是版式语言，不是固定章节模板。

1. **封面**：矛盾命题 + 主体意象 + 三个不同维度的关键数。
2. **统一结论**：唯一研究倾向 + 长短期对照 + 3–4 个验证指标 + 2–3 个核心风险。
3. **业务构成**：两个英雄数字 + 结构图 + 不超过四行的表格 + 结构变化判断。
4. **业务机制**：3–4 个中轴因果节点 + 外围量化证据 + 已验证/待验证或强项/裂缝。
5. **趋势异常**：3–5 年历史主图 + 最新季度/一次性因素侧卡 + 口径解释。
6. **二维财务**：两张回答互补问题的图 + 3–4 个资产负债或投入指标。
7. **利润机制**：时间趋势 + 业务/地区/产品横向比较 + 一句商业解释。
8. **估值约束**：价格或估值位置 + 当前快照 + 2–3 年兑现条件；低分位不等于低风险。
9. **来源与声明**：来源层级 + 数据口径和截止日 + 完整免责声明。

竞争位置可使用横向比较或对照母版；风险路径可使用因果链、对照或趋势异常母版。只选最适合当前命题的关系，不因章节名称固定版式。

### 5. 分配信息预算

#### 三层阅读速度

- **3 秒层**：判断标题、最大数字或关键关系、页底结论。
- **10 秒层**：主图、结构关系和异常批注。
- **30 秒层**：紧凑表格、口径说明、反面论据和来源。

#### P0–P3 优先级

| 优先级 | 内容 | 处理 |
|---|---|---|
| P0 | 判断标题、核心结论 | 证据成立后，不因空间缩小或删除 |
| P1 | 主证据、关键数字、风险/反面论据 | 必须保留，可改变表达方式 |
| P2 | 辅助年份、第二证据、解释性批注 | 空间不足时合并或移页 |
| P3 | 产品全名、次要案例、装饰信息 | 溢出时优先删除 |

信息过载时按 `P3 → P2` 删减，不能缩小 P0/P1 字号。P0 没有证据时撤销判断，不要用排版掩盖证据缺口。

#### 数据页预算

数据结构页优先使用：

1. 两行以内的判断标题。
2. 两个解释“多大”和“怎么变”的英雄数字。
3. 一张主图。
4. 一张不超过四行、只高亮一行的表格。
5. 一句只总结本页证据的页尾判断。

数据过多时删次要年份或合并同类指标，不增加第二张同权重主图。

#### 逻辑页预算

- 一页最多三层或三阶段。
- 每层只保留：阶段标签 + 一个关键数字 + 一条兑现证据 + 一个不确定性。
- 优先表达成熟度、因果或风险等级，产品名称只作证据。
- 连续列举不超过三个产品名或技术缩写。底稿清单超过三项时，**不得原样复制**；改写成“刻蚀能力”“沉积能力”“键合与量检测能力”“封装设备矩阵”等能力组，括号内最多保留两个代表性例子。斜线连接的两个缩写按两项计算；标题、正文、图注和标签都受此限制。

术语压缩示例：

- 错误：`全流程布局：深硅刻蚀、PVD/CVD、电镀、键合、量检测`
- 正确：`封装设备矩阵（示例：深硅刻蚀、键合）`
- 三层仍无法建立主次时，拆页或删除 P2/P3，不缩小字号硬塞。

### 6. 控制整组视觉节奏

- 封面低密度；正文第 1 页（结论页）中高密度；数据证据页高密度；逻辑页中密度；估值和风险页中高密度；来源页文字可多但结构最简单。
- 连续两页不要都使用高密度表格。在数据页之间穿插逻辑、比较或关系页。
- 相邻两页不要用同一种主图重复证明同一结论；后一页补充新的关系维度。
- 标题不超过两行；**封面主标题不超过 15 个字**（含标点符号、数字、百分号）；其他正文页主标题保留”不超过两行”规则，无字数硬限；一个模块不超过三个要点；每页只有一个视觉中心；重点色最多两种；表格只高亮一行。
- 判断标题必须是命题句，不要只写”业务结构””成长性分析”。
- 封面主标题字数超限时，先压缩副词和连词（”的”、”是”、”已经”等），再考虑用符号替代（”+”代替”和/与”，”vs”代替”对比”）；不得通过删掉关键数字或核心矛盾来凑字数。

默认手绘批注语法：

- 下划线 = 主判断；圈选 = 异常值或关键变量；箭头 = 因果、结构迁移或证据接力；虚线 = 阈值、估算区间或未兑现目标。
- 红色 = 风险/恶化/估值透支；绿色 = 优势/改善/已验证；蓝色 = 历史事实/中性数据；黄色 = 新变量/待验证机会。
- 参考图有稳定语义色时可沿用其色板，但整组颜色含义必须一致。
- 同一数据最多使用两种强调方式。

## 输出格式

按以下顺序交付：

1. **元数据**：底稿、受众、页数、画布、当前日期、数据截止日和核心定位。
2. **全局视觉规则**：色板、字体气质、线条、密度节奏和批注语法。
3. **页面规划表**：核心命题、主数据关系、母版、P0/P1 和接力问题。
4. **逐页脚本**：每页按统一六项展开。
5. **制作检查表**：内容、视觉和发布前复核结果。

### 每页八项

每页必须依次写：

1. **页面判断**：一句只判断当前页的命题标题。
2. **标题**：必须单列可直接上屏的标题文本。封面写”系列标题（仅在用户提供时）+ 主标题”，**封面主标题不超过 15 个字**（含标点、数字、百分号）；正文第 1–末页各写一个主标题，保留”不超过两行”规则，无字数硬限。不得把标题藏在页面判断、正文或图表标签中。
3. **可直接上屏内容**：正文和关键数字；标题不在此项重复。
4. **主图数据结构**：图表类型、字段、顺序、完整数列、单位、颜色、高亮点和 A/E/估算/情景标记。
5. **辅图或数据摘要**：补充图表或不超过四项的摘要；写清字段、数字、排列顺序和强调方式。
6. **布局**：只写区块关系、视觉中心和阅读顺序；不得写坐标、像素、百分比、固定宽高或留白数值。
7. **批注与素材**：圈线箭头、图标、主体意象或明确占位，不伪造品牌资产。
8. **页尾判断**：只总结当前页证据；不得提问、预告或承接下一页。

**正文页（封面至倒数第二页）严禁在任何位置写数据来源**，包括"批注与素材""辅图或数据摘要""页尾判断"等模块，均不得出现"数据来源：..."、"（来源：xxx）"、"据xxx"等标注。数据的 A/E/估/情景标记直接写在数字或图例旁即可。所有公司披露、行业数据、市场与预期、新闻公开报道、截止日和免责声明**集中在末页**。末页规则见下文"末页来源与声明规则"。

## 正文第 1 页结论规则（封面不计入页码）

正文第 1 页（即封面后的第一页）按以下层级组织：

1. 顶部：唯一研究倾向，作为第一视觉焦点。
2. 中部：长期逻辑与短期问题的左右对照。
3. 下部左侧：3–4 个跟踪指标；底稿支持时给阈值或观察线，不支持时标“阈值待核验”并写明所缺数据。
4. 下部右侧：2–3 个核心风险。
5. 页底：估值锚及估值与业绩的依赖关系。缺少可靠估值输入时写“估值锚数据不足”，列明当前市值、利润口径或历史分位等缺口，不自行生成价格/倍数。

不要把整篇摘要塞入正文第 1 页；各模块只用于解释顶部结论。

## 末页来源与声明规则

末页必须包含：

1. **公司披露**：年报、季报、公告及其覆盖日期。
2. **行业数据**：机构名、指标和口径日期。
3. **市场与预期**：市场数据、券商一致预期及日期。
4. **新闻与公开报道**：底稿中提到的新闻媒体、行业自媒体等公开信息来源。
5. **数据口径**：实际值 A、预期值 E、估算值、情景假设和数据截止日。
6. **免责声明**：公开资料研究交流，不构成证券、基金或其他金融产品的投资建议、承诺或保证；历史表现与情景推演不代表未来结果，投资者应独立判断并自担风险。

### 来源提取强制规则

- **尽可能多标注**：从底稿全文（包括正文数据旁注、表格脚注、文末来源汇总行）逐条提取所有出现过的来源，不得遗漏。底稿文末若有"数据来源：..."汇总行，必须把汇总行中列出的每个来源都单独列出，不得用"等"字概括。
- **全部来自文档**：所有来源必须严格来自底稿文档原文，**严禁自主发挥或编造**。底稿未提及的机构、媒体、报告一律不得写入末页。
- **分类准确**：每个来源按"公司披露/行业数据/市场与预期/新闻与公开报道"4 类归位，归位有歧义时在来源后用括号注明其覆盖的内容（如"东方财富（行情/财务数据）"）。
- **保留具体日期**：底稿中出现的具体数据日期（如"大基金持股 8.39%（2026-06-23）"、"TrendForce 2026-03-12 报告"）必须原样保留，不得简化为"近期"或"最新"。
- **券商一致预期**：底稿中点名的具体券商（如"华泰、开源、爱建、光大、中邮、国信 6 家券商"）必须逐家列出，不得用"多家券商"概括。
- **来源去重**：同一来源在底稿多处出现时，末页只列一次，但需合并其覆盖的内容范围。

## 投研验证

- 用精确计算工具复核占比、PE、PS、PEG、情景市值和下行空间，不使用心算。
- 对报告关键数据执行抽检；工具把年份、产品代号或术语数字误识别为财务数据时，人工回到原文复核。
- 保留每项核心优势的脆弱性，以及每项风险的对冲论据；压缩时两面都不能删除。
- 严格区分实际值、机构预期、估算和压力测试。
- 最大下行风险在具备可复核利润、估值和情景假设时必须量化，并写明“情景，不是预测”；不得因数字刺眼而删掉底稿已有的下行幅度。输入不足时保留风险页，列出缺口并标记整稿不可发布。
- 发布前复核市场数据时效性、单位、年份、币种和 A/E 标记。

## 最小示例

输入事实：业务 A 占收入 72%、同比 +8%；业务 B 占 18%、同比 +65%；业务 C 尚未形成规模收入。

正确规划：

| 核心命题 | 主数据关系 | 母版 | P0/P1 |
|---|---|---|---|
| 基本盘仍占七成，但新增量来自第二业务 | 部分与整体 | 业务构成 | P0：结构正在变化；P1：72%、18%、+65% |

页面使用“双英雄数字 + 结构条 + 三行表 + 页底判断”。不要把三个业务画成同等大小，也不要把尚未形成收入的业务写成第三增长极。

## 常见错误

| 错误 | 修正 |
|---|---|
| 按底稿目录一章一页 | 先提字段，再按主数据关系选母版 |
| 用像素、比例和坐标锁死布局 | 写清区块关系和视觉中心，让渲染器按内容自适应构图 |
| 把辅图卡片孤立放在页面角落 | 与主图成组排列；图下信息优先使用全宽信息带或并列摘要 |
| 高密度等于文字更多、字号更小 | 保住 P0/P1，先删 P3、再删 P2 |
| 护城河页只画宏大生态图 | 加入量化证据和“已验证/待验证” |
| 成长页罗列产品与缩写 | 归并能力组，突出成熟度差异 |
| 低估值分位等于低风险 | 同时展示未来业绩兑现条件 |
| 风险页只列风险名称 | 写触发条件、对冲证据和量化下行情景 |
| 结论页给出交易动作 | 改写为研究倾向、估值锚和观察条件 |
| 最后一页用总结或互动替代 | 固定输出来源、口径和免责声明 |

- [ ] 默认受众或用户指定受众已明确。

- [ ] 最大下行情景已用可复核数据量化并标注“情景，不是预测”；若无法量化，已列缺口并标记整稿不可发布。
- [ ] 最后一页含来源分类、数据口径、截止日和免责声明。
- [ ] 估值锚和跟踪阈值均有底稿依据；缺失项未被编造，且已写补数要求。
- [ ] 页面规划表标明主数据关系、母版、P0/P1 和页尾判断。
- [ ] 每页都有页面判断、独立标题、上屏内容、主图、辅图或数据摘要、布局、批注和页尾判断八项；封面仅在用户提供时另列系列标题。
- [ ] **封面主标题不超过 15 个字**（含标点、数字、百分号）；超限时已压缩副词连词或用符号替代，未删关键数字或核心矛盾；其他正文页主标题不超过两行，无字数硬限。

- [ ] 布局只描述区块关系；没有坐标、像素、百分比、固定宽高或留白数值；辅图没有孤立角落卡造成的空白失衡。

- [ ] 逻辑页不超过三阶段；已扫描标题、正文、图注和标签中的顿号、斜线、间隔点及逗号术语串，连续产品名或缩写不超过三个，超过时已归并为能力组且例子不超过两个。

- [ ] 全稿没有明确买卖或仓位信号。
- [ ] **末页来源尽可能多标注**，已从底稿全文逐条提取，无遗漏；所有来源均来自底稿原文，未自主发挥或编造；具体日期、券商名单已原样保留，未用"等""多家""近期"概括。
