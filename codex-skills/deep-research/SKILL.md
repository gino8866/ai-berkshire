### 模块一：商业模式与竞争壁垒

**目标**：讲清"这门生意是什么、凭什么能赚钱、护城河深不深"。

#### 1.1 主营业务拆解

- 营收构成**必须用表格**，至少拆到二级业务线：

```markdown
| 产品线/业务 | 营收（亿元） | 占比 | 同比增速 |
|------------|------------|------|---------|
| 业务A | x | x% | +x% |
| 业务B | x | x% | +x% |
| **合计** | **x** | **100%** | **+x%** |
```

- 表格后跟 3-5 条"关键观察"，每条点出一个**结构性事实**（哪块是基本盘、哪块增速最猛、哪块在边缘化），不是复述表格数字。
- 数据来源标注在表格下方（例：数据来源：公司2025年年报）。

#### 1.2 行业地位与市占率

- 给出全球/国内市占率，**注明口径与数据源**（Gartner/Frost & Sullivan/九千咨询等，不同机构口径差异大时要点出"数据罗生门"）。
- 若市占率存在争议，呈现两套数据并说明分歧原因，不要强行选定一个。

#### 1.3 与龙头差距对比（强烈建议）

- **必须有一张与全球/国内龙头的对比表**，维度至少含：收入规模、技术代际/产品宽度、全球份额、核心优势、核心差距。

```markdown
| 维度 | 本公司 | 国际龙头 |
|------|--------|---------|
| 收入规模 | x | x |
| 技术代际 | x | x |
| 全球份额 | x | x |
| 核心优势 | x | x |
```

- 差距分两层写：**差距的本质**（规模/宽度/制程/全球化）+ **差距是否在缩小**（给证据）。

#### 1.4 护城河分析

- 分 2-4 层/维度展开（技术壁垒、客户粘性、政策红利、生态锁定等）。
- **护城河可用 ★ 评级**（★1-5，不含半星）。
- 每条护城河后必须跟**"但需注意其脆弱性"**：什么情况下这条护城河会被突破？（例：AMD授权本质是"借来的地基"；大疆低价+渠道碾压两个季度抢走40%份额）。

> 模块一写作铁律：每一个"优势/壁垒"判断，都必须附带反面论据（"但另一方面…"/"硬币的另一面…"）。不留单边叙事。

### 模块二：财务质量分析

**目标**：用 5 年+ 的财务数据，判断"利润是不是真金白银、资产负债表健不健康"。

#### 2.1 营收与利润趋势

```markdown
| 年度 | 营收（亿元） | 同比增速 | 归母净利（亿元） | 同比增速 | 毛利率 |
|------|------------|---------|----------------|---------|-------|
| 2021 | x | +x% | x | +x% | x% |
| ... | | | | | |
| 2025 | x | +x% | x | +x% | x% |
| 2026Q1 | x | +x% | x | +x% | x% |
```

- 表格后写"关键发现"，重点抓**异常信号**：增收不增利、毛利率趋势性下行、净利增速持续低于营收增速（剪刀差）、扣非与归非的背离。
- 异常信号必须解释**原因**（产品结构变化/新品验证期让利/研发高投入/竞争加剧），不要只描述现象。

#### 2.2 现金流质量

```markdown
| 年度 | 经营现金流净额（亿元） | 归母净利（亿元） | 现金流/净利润 |
|------|----------------------|----------------|-------------|
```

- 现金流/净利润比值是核心质量指标：>1 优秀，0.7-1 合理，<0.5 或负值必须预警。
- 现金流恶化/改善要给原因（合同负债变化、应收账款回收、存货积压）。

#### 2.3 在手订单 / 合同负债 / 存货（设备/制造类必备）

- 合同负债是未来收入的领先指标，存货周转反映健康度。
- 半导体设备类要说明存货周转慢是行业共性（出厂到验收6-18个月），但若存货异常飙升要警惕滞销/减值。

#### 2.4 资产负债表健康度

- 资产负债率、带息负债率、应收账款周转天数、账面现金。
- 与同行对比更有意义（例：资产负债率26.6% vs 同行65%）。
- 应收账款占比过高（>营收40%或周转天数飙升）要单独列为风险点。

#### 2.5 研发投入 / ROE（视行业）

- 研发费用率与全球龙头对比（例：30% vs Lam Research 14%），说明是"前期播种"还是"过度扩张"。
- ROE 偏低要解释原因（IPO募资稀释净资产基数、资产周转率低）。

> 模块二写作铁律：估计/推算值必须标"估算"/"估"；数据不足就写"数据不足"，不要用推测填充确定性。

### 模块三：成长性与估值

**目标**：讲清"未来靠什么增长、行业多大、现在贵不贵、合理区间在哪"。

#### 3.1 未来增长驱动因素

- 用"驱动一/驱动二/驱动三…"结构，每条驱动含：逻辑 + 量化空间（TAM/国产化率提升对应的增量市场）+ 不确定性。
- 驱动要分层：基本盘（确定性高）→ 第二增长曲线（弹性大）→ 远期想象（不确定性高）。

#### 3.2 行业空间测算

```markdown
| 细分市场 | 全球规模 | 中国大陆占比 | 国产化率 | 本公司份额 | 提升空间 |
|---------|---------|------------|---------|---------|---------|
```

- 测算要有数有源（SEMI/Gartner/券商）。
- "国产化率每提升10个百分点对应X亿增量"这类测算要写清计算过程。

#### 3.3 当前估值水平

**三张表必出**：

表1 - 当前估值指标：市值、PE-TTM、PS-TTM、PEG。

表2 - 机构一致预期：未来2-3年预期营收/净利/对应PE（标注券商与口径日期）。

表3 - 同行估值对比：至少3家可比公司（含国际龙头），对比 PE/市值/定位。

> 估值数据必须手算校验，PE = 市值 ÷ 净利润。可用 `tools/financial_rigor.py verify-valuation` 验算。

#### 3.4 合理估值区间判断

- **必须给三情景**：乐观/中性/悲观，每情景含：假设的净利润 → 给予的PE → 合理市值 → 对应股价/较当前空间。
- 给 PEG 视角（PEG = PE ÷ 复合增速），判断估值偏贵/合理/便宜。
- 结论：当前市值**隐含了什么预期**（例：隐含2027年40亿净利、65倍PE），这要求什么增速才能兑现。

### 模块四：风险评估

**目标**：穷举失败路径，量化最大下行风险。

#### 4.1-4.5 多维风险（每个风险独立小节）

每个风险维度必须包含三要素：

1. **风险描述**：具体是什么风险（不要泛泛说"竞争风险"）。
2. **风险等级**：高 / 中高 / 中等 / 中低（用粗体标注）。
3. **反面论据**："但另一方面…"--为什么这个风险可能不会发生、或被对冲。

> 这是本 skill 的核心规则：**没有"但另一方面"的风险分析不合格**。正反两面让读者自己权衡，而非替读者下结论。

#### 4.6 最大的下行风险（必备，单独小节）

- 构造一个**具体的戴维斯双杀场景**：业绩下修 + 估值下修同时发生。
- 量化：在该场景下，PE 从 X 倍回归到 Y 倍，利润降至 Z 亿，对应市值多少，较当前**下行空间 X%-Y%**。
- 极端情景要有数有逻辑，不要只说"可能大跌"。

```markdown
**核心风险场景：{触发因素A} + {触发因素B} + {触发因素C}。**

若 A 发生导致毛利率跌破X%、B 未兑现、叠加 C 冲击--市场定价从"成长股"
切换为"周期股/困境股"，PE 从 {当前} 倍压缩至 {目标} 倍。假设利润恢复至
{N}亿元，对应合理市值 {M}亿元，较当前 {下行百分比}% 的下行空间。
```

- 可选附"乐观情景"对称呈现，避免单边看空。

---

### 模块五：综合结论

**目标**：给读者一个明确的"跟不跟、什么时候跟"的判断。

#### 5.1 核心判断 / 定性判断

- 2-3 段，先讲长期逻辑（为什么是好公司/好赛道），再讲当前问题（估值/盈利质量/结构风险）。
- **不预设立场**：结论从数据自然推出，不预设看多看空。

#### 5.2 是否值得深入跟踪（明确表态）

四档结论，必须选一档并说明理由：
- **绝对值得长期跟踪，但需等待估值回归**（好公司+贵价格）
- **值得跟踪，但当前估值偏贵**（好公司+偏贵）
- **值得观察但当前不宜买入**（有争议+高风险）
- **不建议跟踪**（逻辑破坏）

#### 5.3 关键跟踪指标（编号清单 6-8 条）

- 每条指标写清：**指标名** + **为什么跟踪它** + **看什么信号**（例：毛利率趋势--若跌破40%说明竞争恶化，企稳45%说明仍有定价权）。
- 排序：最核心的指标放第一条（通常是毛利率或核心产品增速）。

#### 5.4 买入时机思考（估值锚）

```markdown
- **理想买入估值**：{X}-{Y}倍PE，对应股价/市值区间
- **可接受买入估值**：{X}-{Y}倍PE，需触发条件
- **需要回避的估值**：超过{X}倍PE，过度透支预期
```

#### 数据来源声明（报告末尾，固定）

```markdown
*数据来源：{公司}{年份}年报、{年份}一季报、{券商}等机构研报、
{Gartner/SEMI/Frost & Sullivan}、{东方财富/同花顺/证券时报}等公开信息。
部分数据根据公开报道估算，已标注"估算"。*
```

## 三、核心写作规则（最高优先级，全程执行）

继承项目投研核心原则，本 skill 强化以下几条：

### 1. 客观、客观、客观

- 严禁"我认为""我觉得""显然"等主观表述，改用"数据显示""证据表明""根据XX来源"。
- 严格区分**事实**（数据支撑）与**观点**（必须标注"观点"/"推测"）。
- 不预设立场：先摆数据 → 再推逻辑 → 最后得结论。

### 2. 呈现正反两面（本 skill 的灵魂）

- **每一个核心判断**都必须附带反面论据（"但另一方面…""硬币的另一面…"）。
- **每一条风险**都必须有"但另一方面"对冲论据。
- 护城河要写脆弱性，优势要写局限性，增长驱动要写不确定性。
- 让读者自己权衡，不替读者下结论。

### 3. 数据纪律

- 所有关键数据标注来源，**关键数据至少2个来源交叉验证**。
- 估计/推算值标注"估算"/"估"，不与披露数据混为一谈。
- 数据不足时诚实写"数据不足"或"不确定"，不用推测填充确定性。
- 市值、PE 等计算数据手算校验，禁止 LLM 心算（可用 `tools/financial_rigor.py`）。

### 4. 量化下行风险

- 每篇报告必须有"最大下行风险"小节，构造戴维斯双杀场景并量化下行空间百分比。
- 不接受"可能下跌""风险较大"这类无量化表述。

### 5. 风格

- 中文，直接、犀利、不说废话。
- 评分用 ★（★1-5，不含半星）。
- 表格呈现关键数据，不要大段堆砌数字。
- 数据来源、风险等级、关键跟踪指标用固定格式。

## 四、执行流程

1. **解析输入**：确定公司全称、代码、板块、研究日期。
2. **手算市值校验**：股价 × 总股本，明确货币单位。
3. **数据收集**（可用后台 Agent 并行）：5年财务、营收结构、市占率、竞争对手、估值、机构预期、风险。按 `skills/financial-data.md` 规范取数，A股主源东方财富+巨潮资讯。
4. **数据交叉验证**：关键数据2源核对，市值/PE 用 `tools/financial_rigor.py` 验算。
5. **按五模块成文**：严格按上述模板，每模块执行其"写作铁律"。
6. **正反两面自查**：通读全文，检查每个核心判断和风险是否都有反面论据；漏的补上。
7. **写入文件**：`reports/{公司名}/{公司名}-deep-research-{YYYYMMDD}.md`（若该公司已有既定报告目录，可沿用原有路径与命名）。
8. **数据抽检**（建议）：按 `skills/investment-research.md` 的数据抽检流程，抽样15%数据点核验偏差≤1%。

## 五、输出要求

1. 五模块结构完整，不可省略模块四的"最大下行风险"和模块五的"关键跟踪指标"+"买入时机"。
2. 每个核心判断与风险均有反面论据。
3. 关键数据有来源，估算值有标注，市值/PE 已校验。
4. 结论明确给出"是否值得跟踪"+"买入估值区间"，不回避表态。
5. 报告末尾附数据来源声明。

## 六、局限性声明

本 skill 产出的是**基于公开资料的深度研究**，不等于投资确定性：
- 公开资料密集的公司（如行业龙头），结论易与市场共识趋同，alpha 有限--需重点做反面检验。
- 数据稀缺或刚上市的公司，部分财务数据需推算，已标注"估算"，读者应通过一手验证（产品体验、供应链访谈、田野调查）补充盲区。
- 估值区间是模型推演，非预言；戴维斯双杀场景是压力测试，非预测。
- 本报告不构成投资建议。
