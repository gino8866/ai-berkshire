# 大模型研究报告

> **研究日期**：2026-07-02
> **研究对象**：基础大模型 / 模型层（闭源 API、开源权重、企业工作流入口）
> **一句话结论**：大模型是文明级技术跃迁，但模型层本身正在快速商品化，真正更好的投资位置通常不在“纯模型公司”，而在“模型+分发”“模型+云”“模型+数据”“模型+企业工作流”。

---

## AI研究偏见自觉

**信息丰富度评级：A 级（信息极多）**

这类题材的最大陷阱不是“没资料”，而是“资料太多，且市场共识太强”。因此本报告刻意把重点放在三个反共识问题：

- 大模型是不是好产品，但未必是好生意？
- 最终利润池会落在模型公司，还是云/分发/数据/应用层？
- 如果能力差距持续缩小，今天给纯模型公司的估值是否在透支 5 年后的利润？

**AI研究置信度**：中高。公开讨论、技术论文、媒体追踪、财报覆盖都很充分。  
**实际投资确定性**：中等。因为行业变化太快，商业模式和监管约束都还在高速变化。

偏见自查：

- [x] 确定性感主要来自商业结构，而不是资料数量
- [x] 不把“模型能力领先”直接等同于“长期定价权领先”
- [x] 不把私募估值直接当成公开市场可承受估值
- [x] 对私有公司财务数据统一按“媒体/第三方估计”处理

## 第一步：核心数据总览

### 1.1 投资逻辑链

```text
知识工作自动化需求上升
  -> 企业愿意为推理/代码/检索/客服付费
    -> 基础模型需求爆发
      -> 训练与推理算力、优质数据、分发入口变成关键瓶颈
        -> 模型公司收入猛增
          -> 但价格竞争、开源追赶、资本开支与监管压力同步上升
            -> 利润池更可能向云、分发、数据、工作流平台倾斜
```

### 1.2 已发生的验证事件

| 验证环节 | 已发生事件 | 说明 |
|---|---|---|
| 企业真实付费 | Ramp AI Index 显示 2026 年 4 月 Anthropic 企业采用率 34.4%，OpenAI 32.3% | 企业已经不是“试点”，而是实打实付费切换 |
| 模型价格下行 | 2026 年论文统计 2020-2026 推理价格约下降 600 倍 | 说明能力在提升，但定价权在变弱 |
| 监管开始介入 | Anthropic 2026-02 更新 RSP 3.0，并把 Risk Report / external review 写入制度 | 前沿模型已从产品竞争走向安全与政策竞争 |
| 大厂把 AI 视为核心资本开支方向 | Google 披露 Gemini 月活超 7.5 亿，且 2026 年继续大幅扩 AI 基建 | 大模型已进入巨头核心预算，而非边缘实验 |

### 1.3 当前行业结构判断

| 环节 | 代表玩家 | 生意特征 | 判断 |
|---|---|---|---|
| 闭源前沿模型 | OpenAI、Anthropic、Google、xAI | 收入增速极快，资本消耗极大 | 产品最性感，财务上最危险 |
| 开源/开放权重 | Meta Llama、DeepSeek、Qwen、Kimi 等 | 直接压缩 API 定价权 | 对行业是利好，对纯闭源模型是压力 |
| 云与算力 | Microsoft、Google、Amazon、CoreWeave | 卖“铲子”，更接近收费站 | 价值捕获更稳 |
| 数据与工作流 | Reddit、Scale AI、Copilot/Claude Code 类入口 | 一旦嵌入流程，粘性更强 | 可能比模型本身更像好生意 |

### 1.4 关键数字快照

| 指标 | 当前观察 | 来源性质 |
|---|---|---|
| Anthropic 估值 / 年化收入 | 约 9650 亿美元 / 470 亿美元，隐含 PS ≈ 20.53x | 媒体报道 + 工具验算 |
| OpenAI 估值 / 年化收入 | 约 8520 亿美元 / 250 亿美元，隐含 PS ≈ 34.08x | 媒体估计 + 工具验算 |
| Ramp 企业采用率 | Anthropic 34.4%，OpenAI 32.3%（2026-04） | 第三方企业支出追踪 |
| Gemini 用户规模 | 月活超 7.5 亿 | 公司披露 |
| 模型推理价格趋势 | 2020-2026 约下降 600 倍 | 学术研究 |

> **段永平式追问**：这门生意到底卖的是什么？  
> 不是“卖参数”，而是卖“把知识工作变成机器流程的能力”。但如果能力越来越接近，真正可收费的就不是模型本身，而是工作流嵌入、数据、分发和默认入口。

## 第二步：生意本质分析

### 一句话定义

大模型行业本质上是在卖“概率性认知劳动力”，把原本由人完成的阅读、写作、编码、检索、客服、分析，变成 API、座席、Agent 或嵌入式功能。

### 这门生意好在哪

| 维度 | 判断 | 解释 |
|---|---|---|
| 需求强度 | 很强 | 企业希望把白领流程自动化，消费端也已养成对话式入口习惯 |
| 边际复制成本 | 理论上低，现实中不低 | 软件复制几乎无限，但高质量推理仍受 GPU、带宽、电力和服务成本约束 |
| 收入扩张速度 | 极强 | 过去 18 个月是企业软件史上最快增速之一 |
| 护城河稳定性 | 中等偏弱 | 能力差距缩小快，客户切换比传统 SaaS 更容易 |
| 资本强度 | 极高 | 训练、推理、数据中心、芯片、人才都极度烧钱 |

### 这门生意坏在哪

1. **模型能力领先未必能沉淀为长期超额利润。**
2. **价格下降速度很可能快于客户粘性形成速度。**
3. **最赚钱的可能是“接住模型的人”，不是“训练模型的人”。**
4. **资本开支前置，利润兑现后置，稍有增速放缓就会被重估。**

### 商业模式拆解

| 模式 | 代表 | 特征 | 长期质量 |
|---|---|---|---|
| API 按量收费 | OpenAI、Anthropic、Google | 容易起量，也最容易打价格战 | 中 |
| 订阅席位 | ChatGPT、Gemini、Claude | 现金流更平滑，但需要持续留存 | 中上 |
| 企业工作流嵌入 | Copilot、Claude Code、Workspace、Agent 平台 | 一旦进入组织流程，迁移成本上升 | 上 |
| 训练数据授权 | Reddit、内容平台 | 稀缺性高，但体量未必最大 | 中上 |
| 云算力打包 | Azure/GCP/AWS | 卖模型也卖算力，双重收费 | 很高 |

> **段永平式追问**：这是一门“对的生意”吗？  
> 如果指“大模型本身”，答案是“产品很好，生意未必最好”；如果指“模型+分发/云/工作流”，答案明显更好。

## 第三步：护城河评估

### 3.1 行业级护城河

| 护城河类型 | 强度 | 结论 |
|---|---|---|
| 品牌 / 心智 | 中 | ChatGPT 最强，Claude 在开发者和企业中变强，Gemini 借 Google 生态追赶 |
| 转换成本 | 中 | 单纯换 API 成本不高；嵌入工作流后的迁移成本开始上升 |
| 网络效应 | 弱到中 | 用户多可带来反馈和分发，但不像支付/社交那样天然闭环 |
| 规模效应 | 强 | 算力采购、推理调度、芯片协同都有规模优势 |
| 技术壁垒 | 中 | 领先是真实的，但领先期显著缩短 |
| 数据壁垒 | 分化 | 通用互联网数据趋于商品化，专有高质量数据更稀缺 |

### 3.2 主要玩家简评

| 公司/阵营 | 护城河 | 证据 | 10 年后还在吗？ |
|---|---|---|---|
| OpenAI | 品牌 + 分发合作 + 消费入口 | ChatGPT 心智最强 | 如果继续只卖模型，护城河会变窄；若拿下系统级入口，则仍强 |
| Anthropic | 企业口碑 + 代码场景口碑 | 企业采用率反超 OpenAI | 更像高端企业软件品牌，仍可能存在，但估值容错率低 |
| Google | 分发 + 云 + 芯片 + 默认入口 | Gemini 用户规模、GCP、TPU、安卓/搜索入口 | 最可能长期保住护城河 |
| Meta 开源路线 | 流量 + 资本 + 开源生态 | Llama 把模型能力向下传导 | 不是直接赚模型钱，但能压缩全行业利润率 |
| DeepSeek / Qwen | 成本效率 + 开源扩散 | 对全球定价权形成持续压力 | 护城河未必在收费，而在“把别人高毛利打掉” |

> **巴菲特式追问**：10 年后护城河还在吗？  
> 纯模型护城河大概率变窄；“模型+默认分发”“模型+企业工作流”“模型+专有数据”的护城河更可能保留下来。

## 第四步：逆向思考与风险清单

### 4.1 失败路径

| 失败路径 | 概率 | 影响 | 预警信号 |
|---|---|---|---|
| 能力继续商品化，API 价格战加速 | 高 | 高 | 开源模型持续逼近闭源，企业多模型切换增多 |
| 资本开支持续飙升但收入增速降档 | 中高 | 高 | 更多融资、更多数据中心承诺，但单位经济没有改善 |
| 监管显著收紧 | 中 | 中高 | 出口管制、强制评估、责任归属增强 |
| 云厂商把模型能力内嵌并吞掉独立模型利润 | 高 | 高 | Google/Microsoft/Amazon 进一步捆绑套餐 |
| 企业更愿意采用开源自部署 | 中 | 中高 | 安全/成本导向客户转向开源 |

### 4.2 最强空方论点

最强空方不是说“大模型没用”，而是说：

1. 大模型**非常有用**，但这不等于纯模型公司**非常赚钱**。
2. 一旦模型能力差距缩小，客户会像采购云实例一样采购模型，拼的是价格、可靠性和集成，而不是神话。
3. 今天私募市场给纯模型公司的估值，默认了它们能同时守住技术领先、品牌领先、企业份额、单位经济和监管许可，这个前提过于完美。

### 4.3 历史类比

- 更像 **云计算早期 + 浏览器入口战 + 电信基础设施** 的混合体，而不是传统软件。
- 终局大概率不是“几十家都高利润”，而是：
  - 少数系统级入口占据心智
  - 云与芯片拿走基础设施利润
  - 一批模型能力成为商品
  - 只有嵌入业务流程的工作流产品保住溢价

> **芒格式追问**：我最可能犯什么错？  
> 最可能犯的错，是把“技术震撼”误判成“商业定价权稳固”。

## 第五步：管理层与组织能力

这部分在行业研究里，不看人格魅力，重点看**资本配置与组织约束**。

| 阵营 | 组织优势 | 隐患 |
|---|---|---|
| OpenAI | 产品节奏快，品牌定义能力强 | 资本需求巨大，治理结构与商业化目标长期仍可能冲突 |
| Anthropic | 企业口碑好，安全叙事清晰，开发者口碑强 | 估值过高后容错率变低，且对外部云/算力依赖仍重 |
| Google | 组织慢，但资产最全 | 创新常被官僚制拖慢，但只要“够好”就能靠分发取胜 |
| Meta | 愿意砸钱，开源心态清晰 | 未必直接赚模型钱，更多是防守和生态控制 |

> **段永平式追问**：如果 CEO 换人，这家公司还能保持竞争力吗？  
> 对纯模型公司，这个问题比传统成熟公司更关键，因为组织仍强依赖前沿研究团队与资本市场信心。

## 第六步：行业与文明趋势

### 6.1 文明趋势判断

**结论：是文明级范式转移，不是短期主题炒作。**

依据不是估值，而是两点：

1. LLM 已经成为新的通用认知接口，正在侵入搜索、办公、编程、客服、内容、科研。
2. 学术与企业证据都指向 AI 采用正在扩散到真实职业任务，而不再停留在演示阶段。

### 6.2 价值链中谁最可能赚钱

```text
电力 / 数据中心 / 芯片
  -> 云平台与模型托管
    -> 基础模型
      -> Agent / Copilot / 企业工作流
        -> 行业应用
```

**最可能稳定赚钱的层**：

1. 云平台
2. 默认分发入口
3. 工作流级产品
4. 稀缺数据资产

**最容易被压缩利润的层**：

1. 纯 API 模型层
2. 缺乏分发入口的独立模型厂商

> **李录式追问**：20 年后回看，模型公司会像标准石油还是 3Com？  
> 更可能少数平台像标准石油，多数纯模型公司像 3Com: 技术重要，但股东未必赚到与想象匹配的钱。

## 第七步：估值与安全边际

### 7.1 估值结论

| 标的层级 | 当前判断 | 原因 |
|---|---|---|
| 纯模型私有巨头（OpenAI / Anthropic） | **大概率不便宜** | 当前估值已隐含极高持续增长和较强利润兑现 |
| 模型+云+分发（Google / Microsoft / Amazon） | **相对合理** | AI 贡献尚未完全反映，但原有现金流能覆盖试错成本 |
| 开源生态防守者（Meta） | **取决于市场对 capex 的容忍度** | 商业化路径还在形成，但其战略地位强 |
| 数据资产受益者（Reddit） | **有继续重估空间** | 市场仍主要按广告公司定价，AI 数据期权未完全计价 |

### 7.2 工具验算记录

| 项目 | 计算式 | 结果 |
|---|---|---|
| Anthropic 隐含 PS | 965 / 47 | 20.53x |
| OpenAI 隐含 PS | 852 / 25 | 34.08x |

说明：以上为基于公开报道估值与年化收入的**近似验算**，不是审计口径财务报表。

### 7.3 三种投资方式

| 方式 | 风险收益特征 | 适合谁 |
|---|---|---|
| 直接买未来模型 IPO | 高波动，高叙事，高回撤风险 | 能承受巨幅波动的人 |
| 买模型生态收费站 | 中高收益，中等风险 | 大多数长期投资者 |
| 等行业洗牌后买胜者 | 较慢，但确定性更高 | 重视赔率和胜率平衡的人 |

> **段永平式追问**：如果股市明天关 5 年，你愿意拿着什么？  
> 我更愿意持有 Google / Microsoft 这类“模型只是现金流增强器”的公司，而不是只能靠下轮融资讲故事的纯模型公司。

## 第八步：最终决策与行动清单

### 8.1 行业总评表

| 维度 | 结论 | 信心度 |
|---|---|---|
| 生意质量（段永平） | 需求极强，但模型层未必是最好的生意 | 高 |
| 护城河（巴菲特） | 护城河在“分发/云/数据/工作流”，不只在模型 | 高 |
| 管理层（段永平+巴菲特） | 纯模型公司高度依赖组织执行与资本市场支持 | 中 |
| 最大风险（芒格） | 商品化 + capex 失控 + 监管收紧 | 高 |
| 文明趋势（李录） | 文明级长期趋势明确 | 高 |
| 估值（巴菲特+段永平） | 纯模型偏贵，生态收费站更可买 | 高 |

### 8.2 策略建议

| 策略 | 建议 |
|---|---|
| 空仓者 | **优先买“模型生态收费站”而不是追纯模型 IPO** |
| 持仓者 | 若持有 Google / Microsoft / Amazon / Meta，可继续持有并观察 AI 变现兑现 |
| 卖出信号 | 纯模型公司上市后若收入增速明显降档、亏损扩大、客户集中度高于预期，应果断降仓 |
| 加仓信号 | 企业工作流渗透继续提升、推理成本继续下降、且云/平台型公司开始兑现 AI 收费利润 |

### 8.3 公开市场优先级

| 优先级 | 标的类型 | 结论 |
|---|---|---|
| 第一优先级 | Google / Microsoft / Amazon 一类 | 最像“低估了 AI 期权的成熟现金流资产” |
| 第二优先级 | Reddit / 数据资产受益者 | 具备 AI 数据稀缺性，可做弹性仓位 |
| 第三优先级 | Meta | 本质是用开源和资本开支压低全行业利润率，值得跟踪 |
| 最后再看 | OpenAI / Anthropic 未来 IPO | 等至少 2-3 个公开季度，看真实亏损率、留存率、客户集中度 |

### 8.4 主题仓位建议

**大模型主题总仓位上限：10% - 15%**

原因：

- 长期趋势足够大，可以配
- 但产业利润分配未定，不能重仓押单一纯模型公司

### 四位大师模拟点评

> **巴菲特**：技术革命不自动等于好投资。你要找的是十年后还能收钱的收费站。

> **芒格**：先反过来想。什么最可能让你亏钱？答案通常不是技术失败，而是你付了一个荒谬的价格。

> **段永平**：模型很好玩，但先问是不是好生意，再问是不是好价格。

> **李录**：这是文明演进的大方向，但大方向正确，不代表每一家参与者都值得买。

## AI分析置信度 vs 投资确定性

- **AI分析置信度**：中高。行业资料与外部验证很多。
- **投资确定性**：中等。因为模型能力、监管、融资环境、成本曲线都还在快速变化。
- **最确定的判断**：大模型会长期存在。
- **最不确定的判断**：今天最火的纯模型公司，5 年后还能否保住利润率与估值。
